# Backup and Restore

Keeping your monitoring data safe is important. This guide explains how to backup your Beszel data and restore it if needed.

## Understanding Beszel Data Storage

Beszel stores its data in specific locations:

### Hub Data

The hub stores:
- Main database (`data.db`)
- Auxiliary database (`auxiliary.db`)
- Encryption keys (`id_ed25519`)
- Backup files (in the `backups/` directory)

### Agent Data

Each agent stores:
- Local cache data
- Temporary files
- Agent-specific configuration

**Important:** The most critical data to backup is the hub data. Agent data can be regenerated by reconnecting agents to the hub.

## Backup Locations

By default, Beszel stores data in:

```
/storage/beszel/
├── data/
│   ├── beszel_data/
│   │   ├── data.db
│   │   ├── auxiliary.db
│   │   ├── id_ed25519
│   │   └── backups/
│   └── agent_data/
└── .env
```

Your actual location may differ based on your `DEFAULT_STORAGE_FOLDER` setting in `.env`.

## Creating Backups

### Manual Backup

The simplest way to backup is to copy the data directory.

**Stop Beszel First:**

```bash
docker-compose down
```

**Copy the Data Directory:**

```bash
# Create a backup directory
mkdir -p /backups/beszel

# Copy the data
cp -r /storage/beszel /backups/beszel/$(date +%Y%m%d)

# Copy the configuration file
cp .env /backups/beszel/$(date +%Y%m%d)/
```

**Restart Beszel:**

```bash
docker-compose up -d
```

### Automated Backup Script

Create a script to automate backups:

```bash
#!/bin/bash
# backup-beszel.sh

BACKUP_DIR="/backups/beszel"
DATE=$(date +%Y%m%d_%H%M%S)
SOURCE_DIR="/storage/beszel"
CONFIG_FILE="/path/to/beszel/.env"

# Create backup directory
mkdir -p "$BACKUP_DIR"

# Stop Beszel
cd /path/to/beszel
docker-compose down

# Create backup
cp -r "$SOURCE_DIR" "$BACKUP_DIR/beszel_$DATE"
cp "$CONFIG_FILE" "$BACKUP_DIR/beszel_$DATE/"

# Restart Beszel
docker-compose up -d

# Remove old backups (keep last 7 days)
find "$BACKUP_DIR" -type d -name "beszel_*" -mtime +7 -exec rm -rf {} \;

echo "Backup completed: beszel_$DATE"
```

Make the script executable:

```bash
chmod +x backup-beszel.sh
```

Add it to crontab for automatic backups:

```bash
# Run daily at 2 AM
0 2 * * * /path/to/backup-beszel.sh
```

### Using the Built-in Backup Feature

Beszel may have built-in backup functionality through the web interface:

1. Log in to the hub
2. Go to Settings or Maintenance
3. Look for a "Backup" option
4. Follow the prompts to create a backup
5. Download the backup file to a safe location

### Backup to Remote Location

For added safety, store backups off-site:

**Using rsync:**

```bash
rsync -avz /storage/beszel user@remote-server:/backups/beszel/
```

**Using scp:**

```bash
scp -r /storage/beszel user@remote-server:/backups/
```

**Using cloud storage:**
- Upload backup files to cloud storage (S3, Google Drive, etc.)
- Use tools like rclone for automated cloud backups

## What to Backup

### Essential Files

You must backup these files:

- `.env` - Configuration file
- `data/beszel_data/data.db` - Main database
- `data/beszel_data/auxiliary.db` - Auxiliary database
- `data/beszel_data/id_ed25519` - Encryption key

### Optional Files

These can be regenerated but are nice to have:

- `data/beszel_data/backups/` - Existing backups
- `data/agent_data/` - Agent data (can be regenerated)

### What Not to Backup

You generally don't need to backup:

- Docker images (can be re-downloaded)
- Container files (recreated on startup)
- Temporary files

## Restoring from Backup

### Restoring the Hub

If you need to restore your hub:

**Stop Beszel:**

```bash
docker-compose down
```

**Restore the Data:**

```bash
# Remove existing data (careful!)
rm -rf /storage/beszel/data

# Restore from backup
cp -r /backups/beszel/20240101/data /storage/beszel/

# Restore configuration
cp /backups/beszel/20240101/.env /path/to/beszel/
```

**Restart Beszel:**

```bash
docker-compose up -d
```

**Verify the Restore:**

1. Check the web interface
2. Verify your servers/agents are listed
3. Check that historical data is present
4. Verify settings are correct

### Restoring to a New Server

If you're moving Beszel to a new server:

1. Install Beszel on the new server following the [Installation Guide](03-installation.md)
2. Stop the new installation: `docker-compose down`
3. Copy your backup to the new server
4. Restore the data as shown above
5. Start Beszel: `docker-compose up -d`
6. Update agent configurations if the hub IP changed

### Partial Restore

If you only need to restore specific files:

**Just the database:**

```bash
docker-compose down
cp /backups/beszel/20240101/data/beszel_data/data.db /storage/beszel/data/beszel_data/
docker-compose up -d
```

**Just the configuration:**

```bash
cp /backups/beszel/20240101/.env /path/to/beszel/
docker-compose restart
```

## Backup Best Practices

### Regular Schedule

- **Daily backups** - For actively used systems
- **Weekly backups** - For less critical systems
- **Before updates** - Always backup before upgrading

### Multiple Backup Locations

- Keep at least one backup on the same server
- Keep one backup on a different server
- Consider off-site or cloud backup for critical data

### Retention Policy

- Keep daily backups for 1 week
- Keep weekly backups for 1 month
- Keep monthly backups for 1 year
- Adjust based on your needs and storage capacity

### Test Your Backups

Regularly test that you can restore from backup:

1. Try restoring to a test environment
2. Verify data integrity
3. Check that everything works
4. Document any issues

### Document Your Backup Process

Keep notes on:
- When backups run
- Where backups are stored
- How to restore from backup
- Any special procedures

### Secure Your Backups

- Encrypt backups if they contain sensitive data
- Restrict access to backup files
- Use secure transfer methods
- Consider backup encryption tools

## Disaster Recovery

### Complete Server Failure

If your hub server fails completely:

1. Set up a new server
2. Install Beszel
3. Restore from your most recent backup
4. Reconnect agents (update HUB_URL if IP changed)
5. Verify everything is working

### Database Corruption

If the database becomes corrupted:

1. Stop Beszel
2. Restore from a backup before the corruption occurred
3. Restart Beszel
4. If corruption persists, try an older backup

### Lost Configuration

If you lose your `.env` file:

1. Restore from backup
2. Or recreate it from `.env.example` and reconfigure
3. You'll need to regenerate agent credentials

## Monitoring Backup Health

### Check Backup Success

Regularly verify that backups are completing:

```bash
# List recent backups
ls -lh /backups/beszel/

# Check backup file sizes
du -sh /backups/beszel/*
```

### Set Up Backup Alerts

Configure alerts to notify you if:
- Backup fails
- Backup location is full
- Backup is too old (not running)

### Monitor Disk Space

Ensure you have enough space for backups:

```bash
df -h /backups
```

## Troubleshooting Backup Issues

### Backup Fails

**Check disk space:**
```bash
df -h
```

**Check permissions:**
Ensure you have permission to read source and write to backup location.

**Check if Beszel is running:**
Stop Beszel before backing up to avoid file conflicts.

### Restore Fails

**Verify backup integrity:**
Check that backup files aren't corrupted.

**Check file permissions:**
Ensure restored files have correct permissions.

**Check configuration:**
Verify `.env` file matches your setup.

### Backup Too Large

**Exclude unnecessary files:**
Don't backup things that can be regenerated.

**Compress backups:**
Use compression to reduce size:
```bash
tar -czf backup.tar.gz /storage/beszel
```

**Reduce retention:**
Keep fewer historical backups.
